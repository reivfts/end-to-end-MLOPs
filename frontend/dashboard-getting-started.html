üìñ How to Use the Dashboard

1. View Service Status

  • Dashboard shows each service with a status indicator
  • üü¢ Green = Service is online
  • üî¥ Red = Service is offline
  • Status updates automatically every 30 seconds

2. Navigate to a Service

  • Click the "Open [Service]" button on any service card
  • Or use the "Quick Links" section at the bottom
  • Each service has its own frontend interface

3. Check Service Information

  • Click on a service name or icon
  • A modal window opens showing:
  • Service description
  • API URL and port
  • Available endpoints
  • HTTP methods (GET, POST, DELETE, etc.)

4. Monitor System Health

  • Scroll to the "System Status" section
  • See all services and their status
  • Click "Check Health" button for manual check
  • Status updates in real-time

5. Access Documentation

  • Quick Links section has documentation links
  • Click to access detailed guides
  • Learn about each service's features
  • View API specifications

‚úÖ Best Practices

Always Use HTTP Server

Don't use file:// protocol. Use Python HTTP server instead:

python -m http.server 8080

This ensures relative links work correctly.

Start Services Before Dashboard

Open the dashboard only after all services are running. This ensures accurate status reporting.

Monitor Status Regularly

Check the System Status section regularly to ensure all services are running correctly.
